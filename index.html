1. WIRELESS REGISTRATION TABLE

Tampilan Wireless Registration Table pada interface Winbox MikroTik.

Pengertian Registration Table

Dari sumber MikroTik Documentation (2024), Registration Table adalah fitur pada MikroTik RouterOS yang menampilkan informasi lengkap tentang klien wireless yang saat ini terhubung ke access point. Fitur ini berfungsi sebagai sistem monitoring yang memungkinkan administrator jaringan untuk memantau status koneksi setiap perangkat, kekuatan sinyal, kualitas koneksi, serta parameter penting lainnya secara real-time.

Dari sumber Network Computing (2023), Registration Table menjadi salah satu alat paling penting untuk troubleshooting masalah wireless karena memberikan visibilitas penuh terhadap metrik koneksi penting seperti signal strength, TX/RX rate, dan uptime dari setiap perangkat yang terhubung ke jaringan wireless MikroTik.

Mengakses Registration Table melalui Winbox

# Langkah-langkah mengakses Registration Table melalui Winbox:
1. Buka aplikasi Winbox dan login ke router MikroTik
2. Pada menu sebelah kiri, pilih Wireless
3. Pilih tab Registration Table
4. Akan muncul daftar klien yang terhubung beserta informasinya
Langkah-langkah mengakses Registration Table melalui interface Winbox.

Mengakses Registration Table melalui CLI

Dari sumber MikroTik Forum (2023), akses melalui Command Line Interface (CLI) memberikan fleksibilitas lebih dalam memantau dan memfilter informasi pada registration table, terutama untuk otomatisasi dan scripting.

/interface wireless registration-table print

Untuk menampilkan informasi lebih detail, gunakan perintah berikut:

2. MONITORING KEKUATAN SINYAL (SIGNAL STRENGTH)

Visualisasi kekuatan sinyal pada perangkat wireless yang terhubung ke MikroTik.

Memahami Signal Strength

Dari sumber MikroTik Troubleshooting Guide (2024), Signal Strength adalah pengukuran kekuatan sinyal wireless yang diterima (dalam dBm) dan merupakan indikator penting untuk mengevaluasi kualitas koneksi wireless. Nilai ini selalu negatif, dan semakin mendekati nol (misalnya -40 dBm), semakin baik kualitas sinyal. MikroTik menggunakan rata-rata tertimbang eksponensial dengan faktor penghalusan 50% untuk mengukur tingkat sinyal.

Dari sumber Networking Best Practices (2023), standar umum untuk interpretasi signal strength adalah:

-30 dBm hingga -50 dBm: Sinyal sangat kuat (excellent)
-50 dBm hingga -60 dBm: Sinyal kuat (good)
-60 dBm hingga -70 dBm: Sinyal cukup baik (fair)
-70 dBm hingga -80 dBm: Sinyal lemah (poor)
Kurang dari -80 dBm: Sinyal sangat lemah, koneksi tidak stabil

Memonitor Signal Strength melalui CLI

Dari sumber MikroTik Forum (2022), untuk memeriksa kekuatan sinyal dari klien tertentu menggunakan CLI, Anda dapat menggunakan perintah berikut:

# Memeriksa signal strength untuk semua klien
/interface wireless registration-table print
# Memeriksa signal strength untuk klien tertentu (berdasarkan indeks)
/interface wireless registration-table get value-name=signal-strength 0
-56dBm@1Mbps
# Memeriksa signal strength untuk klien dengan MAC address tertentu
/interface wireless registration-table get [find where mac-address=AC:84:C6:XX:XX:XX] value-name=signal-strength
-56dBm@1Mbps
# Menampilkan hanya kolom signal-strength untuk semua klien
/interface wireless registration-table print =.proplist=signal-strength
signal-strength: -56dBm@1Mbps
signal-strength: -63dBm@HT20-7

Memonitor Signal Strength dengan Script

Dari sumber MikroTik Script Repository (2023), untuk monitoring berkelanjutan dan pembuatan log, Anda dapat menggunakan script berikut:

# Script untuk mencatat signal strength dan memutuskan klien dengan sinyal lemah
# WiFi monitor script
:local minUptime "15s" # Waktu minimum koneksi
:local minSignalStrength -70 # Batas minimum kekuatan sinyal
/interface wireless registration-table
:foreach i in=[find ap=no] do={
:local signal
:set signal [get $i signal-strength]
:set signal [:pick $signal 0 [:find $signal "dBm"]]
:if ([get $i uptime] > $minUptime && $signal < $minSignalStrength) do={
:log warning ([get $i mac-address] . " was disconnected due to low signal: " . [get $i signal-strength])
/interface wireless registration-table remove $i
:delay 5s
}
}

Script ini dapat dijadwalkan untuk berjalan secara berkala menggunakan Scheduler:

/system scheduler add name="Wireless Signal Monitor" interval=5m on-event=
"/interface wireless registration-table
:foreach i in=[find ap=no] do={
:local signal
:set signal [get $i signal-strength]
:set signal [:pick $signal 0 [:find $signal \"dBm\"]]
:if ([get $i uptime] > \"15s\" && $signal < -70) do={
:log warning ([get $i mac-address] . \" has low signal: \" . [get $i signal-strength])
}
}"

Perhatikan bahwa nilai signal strength pada MikroTik selalu disertai dengan rate data (misalnya "-56dBm@1Mbps"). Untuk keperluan perhitungan atau perbandingan, script harus memisahkan nilai dBm dari rate data tersebut menggunakan fungsi pick dan find.

Kesimpulan Bagian Signal Strength: Dari 2 pendapat tersebut dapat disimpulkan bahwa monitoring signal strength merupakan aspek krusial dalam mengelola jaringan wireless MikroTik. Nilai ideal untuk koneksi stabil adalah -30 dBm hingga -70 dBm, dan dapat dipantau secara efektif melalui CLI, Winbox, atau menggunakan script terjadwal untuk manajemen otomatis.

3. MONITORING KUALITAS KONEKSI (CCQ)

Monitoring Client Connection Quality (CCQ) pada perangkat MikroTik.

Pengertian CCQ (Client Connection Quality)

Dari sumber MikroTik Wiki (2023), CCQ (Client Connection Quality) adalah metrik yang digunakan MikroTik untuk mengukur kualitas koneksi wireless antara access point dan klien. Nilai CCQ dihitung berdasarkan jumlah retransmisi paket dan efisiensi transfer data. Nilai CCQ dinyatakan dalam persentase (0-100%), di mana 100% menunjukkan koneksi yang sempurna tanpa retransmisi.

Dari sumber Wireless Network Engineering (2023), CCQ merupakan indikator yang lebih komprehensif dibandingkan signal strength karena mempertimbangkan berbagai faktor yang mempengaruhi kualitas koneksi seperti interferensi, jarak, dan halangan fisik. Nilai CCQ di bawah 70% umumnya menandakan masalah pada koneksi wireless yang memerlukan investigasi lebih lanjut.

Memonitor CCQ melalui Winbox

Untuk memonitor CCQ melalui Winbox, ikuti langkah-langkah berikut:

# Langkah-langkah memonitor CCQ melalui Winbox:
1. Buka aplikasi Winbox dan login ke router MikroTik
2. Pada menu sebelah kiri, pilih Wireless
3. Pilih tab Registration Table
4. Kolom Tx/Rx CCQ menunjukkan nilai CCQ untuk transmit dan receive
5. Jika kolom ini tidak terlihat, klik kanan pada header tabel dan centang Tx/Rx CCQ

Memonitor CCQ melalui CLI

Dari sumber MikroTik Forum (2022), untuk memeriksa nilai CCQ menggunakan CLI, gunakan perintah berikut:

# Memeriksa CCQ untuk semua klien dengan menampilkan detail lengkap
/interface wireless registration-table print detail
# Output akan mencakup nilai tx-ccq dan rx-ccq
# Memeriksa CCQ untuk klien tertentu (berdasarkan indeks)
/interface wireless registration-table get 0 tx-ccq
72.2
/interface wireless registration-table get 0 rx-ccq
85.6
# Memeriksa CCQ untuk klien dengan MAC address tertentu
/interface wireless registration-table get [find where mac-address=AC:84:C6:XX:XX:XX] tx-ccq
72.2
# Menampilkan hanya informasi CCQ dari registration table
/interface wireless registration-table print =.proplist=mac-address,tx-ccq,rx-ccq
mac-address=AC:84:C6:XX:XX:XX tx-ccq=72.2 rx-ccq=85.6
mac-address=7C:8B:CA:XX:XX:XX tx-ccq=65.7 rx-ccq=79.3

Script Monitoring CCQ

Dari sumber MikroTik Script Collection (2023), berikut adalah script untuk memantau dan mencatat nilai CCQ klien wireless:

# Script untuk monitoring dan logging nilai CCQ
:local minimumCCQ 70 # Batas minimum CCQ yang dapat diterima
/interface wireless registration-table
:foreach i in=[find] do={
:local txccq [get $i tx-ccq]
:local rxccq [get $i rx-ccq]
:local mac [get $i mac-address]
:local interface [get $i interface]
:if ($txccq < $minimumCCQ || $rxccq < $minimumCCQ) do={
:log warning "Poor CCQ detected on $interface: Client $mac has Tx CCQ: $txccq%, Rx CCQ: $rxccq%"
}
}

Untuk menjadwalkan script ini agar berjalan secara otomatis:

/system scheduler add name="CCQ Monitor" interval=10m on-event=
"/interface wireless registration-table
:foreach i in=[find] do={
:local txccq [get $i tx-ccq]
:local rxccq [get $i rx-ccq]
:local mac [get $i mac-address]
:local interface [get $i interface]
:if ($txccq < 70 || $rxccq < 70) do={
:log warning \"Poor CCQ detected on $interface: Client $mac has Tx CCQ: $txccq%, Rx CCQ: $rxccq%\"
}
}"

Nilai CCQ yang rendah secara konsisten (di bawah 60%) dapat mengindikasikan masalah serius seperti interferensi, jarak terlalu jauh, atau perangkat hardware yang bermasalah. Dalam lingkungan produksi, pastikan untuk mengatur notifikasi saat nilai CCQ turun di bawah ambang batas yang dapat diterima.

Kesimpulan Bagian CCQ: Dari 2 pendapat tersebut dapat disimpulkan bahwa Client Connection Quality (CCQ) merupakan metrik penting yang memberikan gambaran komprehensif tentang kualitas koneksi wireless. Monitoring CCQ secara teratur melalui CLI, Winbox, atau script otomatis sangat direkomendasikan untuk memastikan performa jaringan wireless yang optimal dengan nilai ideal di atas 70%.

4. MONITORING TX/RX RATE

Monitoring TX/RX Rate pada perangkat wireless MikroTik.

Pengertian TX/RX Rate

Dari sumber MikroTik Forum (2023), TX/RX Rate mengacu pada kecepatan transfer data antara access point dan klien wireless. TX Rate (Transmit Rate) adalah kecepatan pengiriman data dari access point ke klien, sedangkan RX Rate (Receive Rate) adalah kecepatan penerimaan data dari klien ke access point. Kedua nilai ini diukur dalam Mbps (Megabit per detik) dan dapat bervariasi tergantung pada kualitas sinyal, jenis modulasi, dan kapabilitas perangkat.

Dari sumber Wireless Network Design Guide (2022), TX/RX Rate secara signifikan memengaruhi throughput aktual jaringan wireless. Nilai rate yang rendah biasanya mengindikasikan sinyal yang lemah, interferensi, atau perangkat yang menggunakan standar wireless lama. Monitoring rate ini penting untuk mengidentifikasi perangkat yang mungkin memperlambat keseluruhan jaringan (fenomena yang dikenal sebagai "rate limiting client").

Memonitor TX/RX Rate melalui Winbox

# Langkah-langkah memonitor TX/RX Rate melalui Winbox:
1. Buka aplikasi Winbox dan login ke router MikroTik
2. Pada menu sebelah kiri, pilih Wireless
3. Pilih tab Registration Table
4. Kolom Tx Rate dan Rx Rate menampilkan nilai kecepatan transfer data

Memonitor TX/RX Rate melalui CLI

Dari sumber RouterOS Manual (2024), untuk memeriksa TX/RX Rate menggunakan CLI, gunakan perintah berikut:

# Melihat TX/RX Rate untuk semua klien
/interface wireless registration-table print
# Output mencakup kolom tx-rate dan rx-rate
# Melihat TX/RX Rate untuk klien tertentu (berdasarkan indeks)
/interface wireless registration-table get 0 tx-rate
54Mbps-20MHz/1S
/interface wireless registration-table get 0 rx-rate
54Mbps-20MHz/1S
# Melihat TX/RX Rate untuk klien dengan MAC address tertentu
/interface wireless registration-table get [find where mac-address=AC:84:C6:XX:XX:XX] tx-rate
54Mbps-20MHz/1S
# Menampilkan hanya informasi rate dari registration table
/interface wireless registration-table print =.proplist=mac-address,tx-rate,rx-rate
mac-address=AC:84:C6:XX:XX:XX tx-rate=54Mbps-20MHz/1S rx-rate=54Mbps-20MHz/1S
mac-address=7C:8B:CA:XX:XX:XX tx-rate=26Mbps-20MHz/1S rx-rate=26Mbps-20MHz/1S

Script Monitoring TX/RX Rate

Dari sumber MikroTik Script Collection (2023), berikut adalah script untuk memantau dan mendeteksi klien dengan rate rendah:

# Script untuk mendeteksi klien dengan TX/RX Rate rendah
:local minimumRate "24Mbps" # Batas minimum rate yang dapat diterima
/interface wireless registration-table
:foreach i in=[find] do={
:local txrate [get $i tx-rate]
:local rxrate [get $i rx-rate]
:local mac [get $i mac-address]
:local interface [get $i interface]
:if ([:find $txrate $minimumRate] = 0) do={
:log warning "Low TX Rate detected: Client $mac on $interface has TX Rate: $txrate"
}
:if ([:find $rxrate $minimumRate] = 0) do={
:log warning "Low RX Rate detected: Client $mac on $interface has RX Rate: $rxrate"
}
}

Untuk menjadwalkan monitoring rate secara otomatis:

/system scheduler add name="Rate Monitor" interval=15m on-event=
"/interface wireless registration-table
:foreach i in=[find] do={
:local txrate [get $i tx-rate]
:local rxrate [get $i rx-rate]
:local mac [get $i mac-address]
:local interface [get $i interface]
:if ([:find $txrate \"24Mbps\"] = 0) do={
:log warning \"Low TX Rate detected: Client $mac on $interface has TX Rate: $txrate\"
}
}"

Format TX/RX Rate pada MikroTik mencakup kecepatan data (Mbps), lebar kanal (20MHz/40MHz), dan jumlah spatial stream (/1S, /2S, dll). Misalnya, "54Mbps-20MHz/1S" berarti kecepatan 54 Mbps dengan lebar kanal 20MHz menggunakan 1 spatial stream. Untuk device 802.11n dan yang lebih baru, rate dapat mencapai ratusan Mbps dengan penggunaan multiple spatial streams dan lebar kanal yang lebih besar.

Kesimpulan Bagian TX/RX Rate: Dari 2 pendapat tersebut dapat disimpulkan bahwa TX/RX Rate merupakan parameter penting yang menunjukkan kecepatan transfer data antara access point dan klien. Monitoring TX/RX Rate secara teratur dapat membantu administrator mengidentifikasi perangkat dengan performa rendah yang mungkin memengaruhi keseluruhan kinerja jaringan wireless.

5. MONITORING STATUS WIRELESS SECARA MENYELURUH

Dashboard monitoring status wireless secara menyeluruh pada MikroTik RouterOS.

Tools Monitoring Bawaan MikroTik

Dari sumber MikroTik Documentation (2024), selain registration table, MikroTik RouterOS menyediakan beberapa tools bawaan untuk monitoring status wireless secara menyeluruh, yang dapat membantu administrator dalam memahami kondisi jaringan secara real-time dan historical.

# Tools monitoring wireless pada MikroTik:
1. Wireless Snooper: Memantau penggunaan frekuensi sekitar dan menampilkan perangkat yang menempati setiap frekuensi
2. Wireless Scanner: Memindai dan menampilkan informasi tentang access point di sekitar
3. Spectral Scan: Memantau data spektral secara kontinu untuk menganalisis interferensi
4. Torch: Memonitor trafik secara real-time pada interface wireless
5. Traffic Monitor: Memantau throughput pada interface wireless

Menggunakan Wireless Snooper

Dari sumber MikroTik Wiki (2023), Wireless Snooper adalah alat yang sangat berguna untuk memantau penggunaan frekuensi di sekitar dan mengidentifikasi potensial interferensi atau kepadatan kanal. Tool ini tersedia di CLI dan Winbox.

# Menggunakan Wireless Snooper melalui CLI
/interface wireless snooper
# Atau untuk interface WiFi baru:
/interface wifi flat-snoop

2412 AC:84:C6:XX:XX:XX - -61 -96 35 -
2437 F4:F5:D8:XX:XX:XX (ap) Neighbor-WiFi -65 -96 31 -

Menggunakan Wireless Scanner

Dari sumber RouterOS Manual (2023), Wireless Scanner digunakan untuk memindai dan menampilkan informasi tentang access point di sekitar, termasuk SSID, sinyal, channel, dan mode keamanan.

# Menggunakan Wireless Scanner melalui CLI
/interface wireless scan wlan1
# Atau untuk interface WiFi baru:
/interface wifi scan wifi1
ADDRESS SSID FREQUENCY SIG... RADIO-NAME
E4:8D:8C:XX:XX:XX MyNetwork 2412MHz -56 MT920
F4:F5:D8:XX:XX:XX Neighbor-WiFi 2437MHz -65 MT760

Monitoring Trafik dengan Torch

Dari sumber MikroTik Network Monitoring (2022), Torch adalah alat bawaan MikroTik yang sangat berguna untuk memonitor trafik secara real-time pada interface, termasuk interface wireless.

# Menggunakan Torch untuk monitoring trafik wireless
/tool torch wlan1
TX RX
PACKET IP SRC DST-MAC PACKET IP SRC DST-MAC
153kbps 192.168.88.10 AC:84:C6:XX:XX 42kbps 192.168.88.1 E4:8D:8C:XX:XX

Script Monitoring Menyeluruh

Dari sumber MikroTik Script Repository (2023), berikut adalah script komprehensif untuk monitoring dan logging status wireless secara keseluruhan:

# Script comprehensive monitoring wireless status
:local logFileName "wireless-status"
/interface wireless registration-table
:log info "===== WIRELESS STATUS REPORT $[/system clock get date] $[/system clock get time] ====="
# Log client count
:local clientCount [/interface wireless registration-table print count]
:log info "Total wireless clients connected: $clientCount"
# Log status per client
:foreach i in=[find] do={
:local mac [get $i mac-address]
:local interface [get $i interface]
:local signal [get $i signal-strength]
:local txrate [get $i tx-rate]
:local rxrate [get $i rx-rate]
:local uptime [get $i uptime]
:local txccq [get $i tx-ccq]
:local rxccq [get $i rx-ccq]
:log info "Client: $mac on $interface - Signal: $signal, TX/RX Rate: $txrate/$rxrate, CCQ: $txccq%/$rxccq%, Uptime: $uptime"
:if ([:pick $signal 0 [:find $signal "dBm"]] < -70) do={
:log warning "Poor signal for client $mac: $signal"
}
:if ($txccq < 70 || $rxccq < 70) do={
:log warning "Poor CCQ for client $mac: TX/RX CCQ = $txccq%/$rxccq%"
}
}

Untuk menjadwalkan monitoring status secara berkala:

/system scheduler add name="Wireless Status Monitor" interval=30m on-event=
"/interface wireless registration-table
:log info \"===== WIRELESS STATUS REPORT $[/system clock get date] $[/system clock get time] =====\"
:local clientCount [print count]
:log info \"Total wireless clients: $clientCount\"
:foreach i in=[find] do={
:local mac [get $i mac-address]
:local signal [get $i signal-strength]
:local txccq [get $i tx-ccq]
:log info \"Client $mac - Signal: $signal, CCQ: $txccq%\"
}"

Dalam jaringan wireless yang sibuk, log yang dihasilkan oleh script monitoring menyeluruh dapat tumbuh dengan cepat. Pertimbangkan untuk menggunakan external logging server atau secara berkala membersihkan log yang sudah lama untuk menghindari penggunaan disk yang berlebihan pada router MikroTik.

Kesimpulan Bagian Monitoring Menyeluruh: Dari 2 pendapat tersebut dapat disimpulkan bahwa MikroTik RouterOS menyediakan berbagai tools dan mekanisme untuk monitoring status wireless secara komprehensif. Dengan mengkombinasikan tools bawaan seperti Wireless Snooper, Scanner, dan Torch, serta menggunakan script otomatis, administrator dapat memperoleh gambaran lengkap tentang kondisi jaringan wireless dan mengidentifikasi masalah potensial sebelum mempengaruhi pengguna.
